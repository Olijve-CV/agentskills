# Workflow Reference

## Defaults

- Language: Chinese
- Format: Douyin / Video Account short video
- Aspect ratio: `9:16`
- Duration target: about 2 minutes
- Style: core-points commentary

## Recommended Source Mix

Start from the user link, then improve the source set with 2 to 4 stronger sources when possible:

- official or publisher page
- product / catalog page with structured metadata
- review aggregate or reading community page
- one longer review, interview, or reading note with distinct framing

Use the strongest overlapping claims, not the loudest single source.

## Scene Design Rules

- Aim for 6 to 8 scenes.
- Keep most scenes between 10 and 20 seconds.
- Start with tension, surprise, or a strong promise.
- Keep one scene to one main point.
- Subtitles should be concise enough to read on a phone.
- Image prompts should describe composition, tone, and symbolism, not just keywords.

## Image Generation Notes

- The CLI uses `chatgpt-web-cdp` and expects a signed-in browser over CDP.
- Default image page: `https://chatgpt.com/images/`
- Prefer prompts that avoid embedded text, UI mockups, or tiny details.
- If several scenes should look visually consistent, repeat the same style cues across prompts.

## Voiceover Notes

- `edge-tts` is the default engine.
- Recommended default voice: `zh-CN-XiaoxiaoNeural`
- Speed can usually stay at `+0%` to `+8%` for short-form book videos.
- If the wording sounds stiff, rewrite the narration instead of over-tuning TTS.

## BGM Notes

Automatic BGM selection is local-library based. The CLI scans filenames under `--bgm-dir` and scores them against:

- `bgm.query`
- `bgm.mood_tags`
- scene energies

Good filename examples:

- `motivational_piano_uplifting.mp3`
- `calm_inspiring_corporate.wav`
- `dramatic_pulse_high-energy.mp3`

If the library is weakly named, the selection quality will also be weak.

## Rendering Requirements

- install dependencies in `skills/book2video/remotion` with `npm install`
- Remotion is the default visual renderer
- `ffmpeg` is still required by the render toolchain, either directly for the fallback renderer or indirectly by Remotion rendering
- the title card and scene overlays are generated by the Remotion composition
- the older `book2video.mjs render` command remains as a fallback path

## Troubleshooting

- If image generation stalls, verify the CDP browser is signed in to `chatgpt.com`.
- If TTS fails, run `node .\skills\book2video\scripts\book2video.mjs list-voices` and confirm `edge-tts` works locally.
- If Remotion render fails before launch, run `npm install` in `skills\book2video\remotion`.
- If render fails immediately, the most likely cause is missing `ffmpeg` or a broken Remotion install.
- If subtitles render but Chinese text looks wrong, install or point `ffmpeg` to a CJK-capable font such as Microsoft YaHei.
